# AGENTS.md — JavaCareerPlan 项目规范

## 项目简介
这是一个Java后端学习计划项目，包含12周的学习内容。本文件定义了生成详解时的规范，确保每次对话都能遵循一致的格式和存放位置。

---

## 详解文件生成规则

### 1. 存放位置
所有详解文件必须存放在**独立的子目录**中，格式如下：

```
Week{NN}-{主题}/
    Day{NN}-{主题名}/
        {主题名}详解.md      ← 详解文件
    Day{NN}-{主题名}.md      ← 日计划文件（可选）
```

**示例**：
```
Week01-Java基础/
    Day01-ArrayList与LinkedList/
        ArrayList与LinkedList详解.md
    Day02-HashMap底层原理/
        HashMap底层原理详解.md
    Day03-ConcurrentHashMap/
        ConcurrentHashMap原理详解.md
    Day04-LinkedHashMap与LRU缓存/
        LinkedHashMap与LRU缓存详解.md
```

**注意事项**：
- 详解文件必须放在子目录中，不能与日计划文件平级
- 子目录名必须与日计划文件名保持一致（不含 `.md` 后缀）

### 2. 命名规则
- **子目录名**：`Day{NN}-{主题名}`（与日计划文件同名）
- **详解文件名**：`{主题名}详解.md` 或 `{主题名}原理详解.md`

### 3. 格式规范

#### 3.1 标题格式
```markdown
# Day{NN} 详解 — {主题名}
```

#### 3.2 固定章节结构
```markdown
## 一、{知识点A}
### 1.1 小节标题
- 理论讲解
- 代码示例（```java 代码块）

## 二、{知识点B}
### 2.1 小节标题
...

## N、LeetCode题目解析
- 题目描述
- 思路分析
- Java代码
- 复杂度分析

## N+1、今日学习要点总结
- 核心要点归纳
- 验收清单（- [ ] checkbox）

## N+2、练习建议
...

## N+3、扩展阅读
...
```

#### 3.3 内容特征
- 使用 `**加粗**` 标记关键概念
- 使用 `✅` / `❌` 标记正确/错误写法
- 使用表格对比特性
- 使用 ASCII 图表展示数据结构
- 章节间用 `---` 分隔线
- 代码注释使用中文
- LeetCode题目必须添加超链接，格式为：`[题号. 题目名称（难度）](https://leetcode.cn/problems/题目slug/)`
- LeetCode题目使用中文链接

### 4. 生成流程
当用户请求生成某天的详解时：
1. 读取对应的日计划文件（如 `Day{NN}-{主题名}.md`）
2. 在 `Week{NN}-{主题}/` 下创建 `Day{NN}-{主题名}/` 子目录（如果不存在）
3. 生成详解文件 `Day{NN}-{主题名}/{主题名}详解.md`
4. 遵循上述格式规范

### 5. 质量检查
生成详解后，检查以下内容：
- [ ] 文件是否存放在正确的子目录中
- [ ] 文件名是否符合命名规则
- [ ] 标题格式是否正确
- [ ] 是否包含所有必要的章节
- [ ] 代码块是否使用 ```java 标记
- [ ] 是否包含LeetCode题目解析
- [ ] 是否包含验收清单
- [ ] LeetCode题目是否都添加了跳转超链接

## 其他规范

### 日计划文件
- 使用 checkbox 清单格式
- 包含学习目标、LeetCode题目链接、学习笔记区域

### 代码规范
- 代码块使用 ```java 标记
- 代码注释使用中文
- 关键代码添加行内注释

### 文档规范
- 使用 Markdown 格式
- 章节使用中文数字（一、二、三...）
- 子章节使用阿拉伯数字（1.1、1.2...）
