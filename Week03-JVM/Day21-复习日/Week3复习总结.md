## 三、垃圾回收速记

### 3.1 四种回收算法对比

| 算法 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| **标记-清除** | 标记后清除 | 实现简单 | 碎片多 | 老年代(CMS) |
| **复制算法** | 复制存活对象 | 无碎片 | 浪费内存 | 新生代 |
| **标记-整理** | 标记后整理 | 无碎片 | 效率低 | 老年代 |
| **分代收集** | 分代采用不同算法 | 综合最优 | 实现复杂 | JVM实际使用 |

### 3.2 GC Roots速记

```
GC Roots（5种）：
┌─────────────────────────────────────────────────────────┐
│  1. 虚拟机栈中的引用（局部变量）                         │
│  2. 方法区中的静态变量引用                               │
│  3. 方法区中的常量引用                                   │
│  4. 本地方法栈中JNI引用                                 │
│  5. 被synchronized持有的对象                            │
└─────────────────────────────────────────────────────────┘
```

### 3.3 三种GC对比

| GC类型 | 区域 | 触发条件 | 特点 |
|--------|------|---------|------|
| **Minor GC** | 新生代 | Eden区满 | 频繁、快速 |
| **Major GC** | 老年代 | 老年代空间不足 | 较慢 |
| **Full GC** | 整个堆 | 多种条件 | 最慢，应避免 |

## 四、收集器速记（CMS、G1、ZGC）

### 4.1 收集器演进时间线

```
收集器演进：

JDK 1.4 (2002)  →  JDK 9 (2017)  →  JDK 14 (2019)  →  JDK 21 (2023)
     │                │                │                 │
     ▼                ▼                ▼                 ▼
   引入CMS        G1成为默认        CMS废弃           ZGC分代化

当前推荐（2024+）：
├── 通用场景：G1（JDK9+默认）
├── 大堆 + 低延迟：ZGC（JDK21+推荐）
└── 高吞吐量：Parallel Scavenge
```

### 4.2 CMS四阶段（已废弃，JDK14）

```
CMS工作流程：

┌─────────────────────────────────────────────────────────┐
│  阶段1: 初始标记 (Initial Mark) - STW                   │
│  - 标记GC Roots直接关联的对象                           │
│  - 速度很快                                              │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  阶段2: 并发标记 (Concurrent Mark) - 与用户线程并发     │
│  - 遍历对象图                                           │
│  - 耗时最长，但不需要STW                                │
│  - 可能产生浮动垃圾                                      │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  阶段3: 重新标记 (Remark) - STW                         │
│  - 修正并发标记期间的变化                                │
│  - 比初始标记稍长                                       │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  阶段4: 并发清除 (Concurrent Sweep) - 与用户线程并发   │
│  - 清除可回收对象                                        │
│  - 使用标记-清除算法，产生碎片                           │
└─────────────────────────────────────────────────────────┘
```

### 4.2 CMS vs G1对比

| 对比项 | CMS | G1 |
|--------|-----|-----|
| **收集范围** | 老年代 | 整堆 |
| **内存布局** | 连续内存 | Region化 |
| **算法** | 标记-清除 | 标记-整理+复制 |
| **碎片** | 有 | 无 |
| **停顿** | 不可预测 | 可预测 |
| **适用堆** | < 4GB | > 4GB |
| **JDK** | JDK9前 | JDK9+默认 |

### 4.3 G1 Region类型

```
G1 Region类型：

┌────┬────┬────┬────┬────┐
│ E  │ E  │ S  │ O  │ O  │  E = Eden
├────┼────┼────┼────┼────┤  S = Survivor
│ O  │ O  │ H  │ H  │    │  O = Old
├────┼────┼────┼────┼────┤  H = Humongous（大对象）
│    │    │    │    │    │  空 = 空闲
└────┴────┴────┴────┴────┘
```

### 4.4 ZGC速记（JDK21+推荐）

```
ZGC核心特点：

┌─────────────────────────────────────────────────────────┐
│  停顿时间：    < 1ms（不随堆大小增加）                   │
│  最大堆：      支持TB级（理论16TB+）                     │
│  分代模式：    JDK21引入，JDK23默认                      │
│  着色指针：    使用64位指针存储元数据                     │
│  适用场景：    大堆、低延迟要求高的应用                   │
└─────────────────────────────────────────────────────────┘

ZGC演进：
JDK 11 (2018) → JDK 15 (2020) → JDK 21 (2023) → JDK 23 (2024)
     │               │               │               │
     ▼               ▼               ▼               ▼
  实验性ZGC      生产就绪ZGC      分代ZGC         默认分代
```

### 4.5 收集器选择速记

| 场景 | 推荐收集器 | 参数 |
|------|-----------|------|
| **通用** | G1 | -XX:+UseG1GC |
| **大堆+低延迟** | ZGC | -XX:+UseZGC |
| **极致低延迟** | ZGC | -XX:+UseZGC -XX:MaxGCPauseMillis=1 |
| **高吞吐量** | Parallel | -XX:+UseParallelGC |
| **小堆** | Serial | -XX:+UseSerialGC |

      └───── 连接 ─────┘

### 5.2 双亲委派流程

```
双亲委派：

Custom ClassLoader
        │ 委派
        ▼
Application ClassLoader
        │ 委派
        ▼
Extension ClassLoader
        │ 委派
        ▼
Bootstrap ClassLoader（尝试加载）
        │
        ├── 能加载 → 返回Class
        │
        └── 不能加载 → 委派给子加载器
```

| 场景 | 原因 | 解决方案 |
|------|------|---------|
| **SPI** | 核心API加载第三方实现 | 线程上下文类加载器 |
| **Tomcat** | Web应用隔离 | WebAppClassLoader |
| **热部署** | 动态替换类 | 自定义类加载器 |

### 6.1 OOM排查流程

```
OOM排查：

1. 获取异常信息（OOM类型、堆栈）
         │
         ▼
2. 查看GC日志（GC频率、耗时）
         │
         ▼
3. 导出堆转储（jmap -dump）
         │
         ▼
4. 分析堆转储（MAT/VisualVM）
         │
         ▼
5. 定位代码问题（内存泄漏/溢出）
```

### 6.2 常用命令速查

```bash
# 进程查看
jps -l

# GC监控
jstat -gcutil <pid> 1000 10

# 线程分析
jstack <pid>

# 堆转储
jmap -dump:format=b,file=heap.hprof <pid>

# Arthas
dashboard    # 实时面板
thread -b    # 死锁检测
watch        # 方法监控
trace        # 调用链路
jad          # 反编译
```

### 6.3 常用JVM参数速查

```bash
# 堆内存
-Xms4g -Xmx4g -Xmn2g

# 元空间
-XX:MetaspaceSize=256m -XX:MaxMetaspaceSize=512m

# GC
-XX:+UseG1GC -XX:MaxGCPauseMillis=200

```

## 七、常见面试题汇总

### 7.1 JVM内存模型相关

**Q1: JVM运行时数据区有哪些？**

A: 分为线程私有和线程共享两类：
- 线程私有：程序计数器、虚拟机栈、本地方法栈
- 线程共享：堆、方法区（元空间）

**Q2: 堆内存的分代结构是怎样的？**

A: 堆分为新生代和老年代，比例默认1:2。新生代分为Eden区和两个Survivor区（S0、S1），比例默认8:1:1。

**Q3: 为什么JDK8用元空间替代永久代？**

A: 永久代大小固定，容易OOM；元空间使用本地内存，默认只受物理内存限制，更灵活。

### 7.2 垃圾回收相关

**Q4: 如何判断对象是否可回收？**

A: JVM使用可达性分析算法，从GC Roots出发，不可达的对象可以被回收。

**Q5: GC Roots有哪些？**

A: 虚拟机栈引用、静态变量、常量、JNI引用、同步锁持有的对象。

**Q6: 新生代为什么用复制算法？**

A: 新生代对象存活率低，复制算法效率高，且没有内存碎片。

### 7.3 收集器相关

**Q7: CMS的四个阶段是什么？（注意：CMS已在JDK14废弃）**

A: 初始标记（STW）→ 并发标记 → 重新标记（STW）→ 并发清除

**Q8: G1的特点是什么？**

A: Region化内存、可预测停顿、整堆收集、无内存碎片。

**Q9: ZGC的特点是什么？（面试高频）**

A: 停顿时间<1ms、支持TB级堆、使用着色指针、JDK21+推荐使用。

**Q10: 收集器如何选择？**

A: 通用场景用G1，大堆+低延迟用ZGC，高吞吐用Parallel，CMS已废弃不推荐。

**Q11: CMS为什么被废弃？**

A: CMS有浮动垃圾、内存碎片、CPU敏感等缺点，已被G1和ZGC替代。

### 7.4 类加载相关

**Q10: 什么是双亲委派？**

A: 类加载时先委派给父类加载器，父类无法加载时自己才加载。

**Q11: 双亲委派的好处？**

A: 避免类重复加载，保证核心API安全。

**Q12: 如何打破双亲委派？**

A: 重写loadClass()、使用线程上下文类加载器（SPI）、Tomcat的WebAppClassLoader等。

### 7.5 调优相关

**Q13: 线上OOM如何排查？**

A: 1.获取异常信息 → 2.查看GC日志 → 3.导出堆转储 → 4.使用MAT分析 → 5.定位代码问题

**Q14: 常用的JVM调优参数有哪些？**

A: -Xms/-Xmx（堆大小）、-Xmn（新生代）、-Xss（栈大小）、-XX:+UseG1GC（G1收集器）
