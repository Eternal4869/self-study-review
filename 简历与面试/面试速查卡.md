# Java面试速查卡

## 🔥 高频面试题速答

### 1. HashMap底层原理
**数据结构：** 数组 + 链表 + 红黑树
**put()流程：**
1. 计算key的hash值（高16位异或低16位）
2. 定位数组位置（hash & (n-1)）
3. 如果为空，直接插入
4. 如果不为空，判断key是否相同
5. 相同则覆盖value
6. 不同则判断是链表还是红黑树
7. 链表长度>=8且数组长度>=64，转红黑树
8. 插入后判断是否需要扩容

### 2. synchronized锁升级
**偏向锁 → 轻量级锁 → 重量级锁**
- 偏向锁：只有一个线程访问，CAS记录线程ID
- 轻量级锁：有少量竞争，自旋+CAS
- 重量级锁：竞争激烈，阻塞等待

### 3. 线程池execute()流程
1. 当前线程数 < corePoolSize → 创建核心线程
2. 核心线程满 → 放入workQueue
3. workQueue满 → 创建非核心线程
4. 线程数 > maximumPoolSize → 执行拒绝策略

### 4. volatile原理
- 保证可见性：修改后立即刷新到主内存
- 保证有序性：禁止指令重排序
- 不保证原子性：i++不是原子操作

### 5. Spring IOC启动流程
1. 创建BeanFactory
2. 加载配置文件
3. 创建BeanDefinition
4. 实例化Bean
5. 属性注入
6. 初始化Bean
7. 放入单例池

### 6. Spring Bean生命周期
实例化 → 属性注入 → Aware接口 → BeanPostProcessor → InitializingBean → init-method → 使用 → DisposableBean → destroy-method

### 7. Spring Boot自动配置原理
1. @SpringBootApplication → @EnableAutoConfiguration
2. @EnableAutoConfiguration → @Import(AutoConfigurationImportSelector)
3. AutoConfigurationImportSelector → 读取spring.factories
4. 根据条件注解（@ConditionalOnClass等）过滤配置类

### 8. MySQL索引原理
- B+树结构：叶子节点存储数据，非叶子节点存储索引
- 聚簇索引：叶子节点存储完整数据行
- 非聚簇索引：叶子节点存储主键值（回表）
- 覆盖索引：查询字段都在索引中，无需回表

### 9. Redis缓存穿透/击穿/雪崩
- 穿透：查询不存在的数据 → 布隆过滤器、缓存空值
- 击穿：热点key过期 → 互斥锁、逻辑过期
- 雪崩：大量key同时过期 → 随机过期时间、集群

### 10. Kafka消息不丢失
- 生产者：acks=all， retries>0
- Broker：replication.factor>1，min.insync.replicas>1
- 消费者：手动提交offset，幂等消费
