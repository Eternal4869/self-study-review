# Day08 详解 — 线程基础

## 一、线程创建方式

### 1.1 四种线程创建方式对比

| 创建方式 | 优点 | 缺点 | 适用场景 |
|---------|------|------|---------|
| **Thread类** | 简单直接 | 无法继承其他类 | 简单任务 |
| **Runnable接口** | 解耦任务与线程，支持多实现 | 无返回值 | 通用场景 |
| **Callable接口** | 有返回值，可抛异常 | 相对复杂 | 需要返回结果 |
| **线程池** | 资源复用，管理方便 | 配置复杂 | 生产环境 |

### 1.2 Thread类创建

```java
// 方式一：继承Thread类
public class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread方式: " + Thread.currentThread().getName());
    }
}

// 使用
MyThread thread = new MyThread();
thread.start();  // 必须调用start()，不能直接调用run()
```

### 1.3 Runnable接口创建

```java
// 方式二：实现Runnable接口
public class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("Runnable方式: " + Thread.currentThread().getName());
    }
}

// 使用
Thread thread = new Thread(new MyRunnable());
thread.start();

// Java 8+ Lambda写法
Thread lambdaThread = new Thread(() -> {
    System.out.println("Lambda方式: " + Thread.currentThread().getName());
});
```

### 1.4 Callable接口创建

```java
// 方式三：实现Callable接口（有返回值）
public class MyCallable implements Callable<Integer> {
    @Override
    public Integer call() throws Exception {
        System.out.println("Callable方式: " + Thread.currentThread().getName());
        return 1 + 1;  // 返回结果
    }
}

// 使用FutureTask获取结果
FutureTask<Integer> futureTask = new FutureTask<>(new MyCallable());
Thread thread = new Thread(futureTask);
thread.start();

// 阻塞等待获取结果
Integer result = futureTask.get();  // 结果为2
```

### 1.5 线程池创建

```java
// 方式四：使用线程池
ExecutorService executor = Executors.newFixedThreadPool(5);

// 提交Runnable任务
executor.submit(() -> {
    System.out.println("线程池方式: " + Thread.currentThread().getName());
});

// 提交Callable任务
Future<Integer> future = executor.submit(() -> {
    return 1 + 1;
});

executor.shutdown();
```

## 二、线程状态详解

### 2.1 六种线程状态

```
        ┌─────────────────┐
        │      NEW        │  新建状态
        │  (new Thread()) │
        └────────┬────────┘
                 │ start()
                 ▼
        ┌─────────────────┐
        │    RUNNABLE     │  可运行状态
        │  (就绪+运行中)   │
        └───┬─────────┬───┘
            │         │
  wait()    │         │   sleep()时间到
  join()    │         │   wait(time)时间到
            ▼         ▼
    ┌───────────┐ ┌───────────────┐
    │  WAITING  │ │TIMED_WAITING  │
    │ (无限等待) │ │  (超时等待)    │
    └─────┬─────┘ └───────┬───────┘
          │               │
          │ notify()      │ 时间到
          ▼               │
    ┌─────────────────────┘
    │
    ▼
┌─────────────────┐
│    RUNNABLE     │  回到可运行状态
└─────────────────┘

    ┌─────────────────┐
    │    BLOCKED      │  阻塞状态
    │  (等待获取锁)    │
    └─────────────────┘
            │
            │ 获取到锁
            ▼
    ┌─────────────────┐
    │    RUNNABLE     │
    └─────────────────┘

所有状态最终 →
    ┌─────────────────┐
    │   TERMINATED    │  终止状态
    └─────────────────┘
```

### 2.2 状态说明

| 状态 | 说明 | 触发条件 |
|------|------|---------|
| **NEW** | 新建但未启动 | `new Thread()` |
| **RUNNABLE** | 可运行（包括就绪和运行中） | `thread.start()` |
| **BLOCKED** | 阻塞，等待获取synchronized锁 | 等待synchronized锁 |
| **WAITING** | 无限等待 | `Object.wait()`、`Thread.join()`、`LockSupport.park()` |
| **TIMED_WAITING** | 超时等待 | `Thread.sleep(ms)`、`Object.wait(ms)`、`Thread.join(ms)` |
| **TERMINATED** | 终止 | `run()`方法执行完毕或异常退出 |

### 2.3 状态验证代码

```java
public class ThreadStateTest {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });

        // NEW状态
        System.out.println("1. NEW: " + thread.getState());  // NEW

        thread.start();

        // TIMED_WAITING状态
        System.out.println("2. TIMED_WAITING: " + thread.getState());  // TIMED_WAITING

        thread.join();
        
        // TERMINATED状态
        System.out.println("3. TERMINATED: " + thread.getState());  // TERMINATED
    }
}
```

## 三、start() vs run()区别

### 3.1 核心区别

| 对比项 | start() | run() |
|--------|---------|-------|
| **线程创建** | 创建新线程 | 在当前线程执行 |
| **并发性** | 异步执行 | 同步执行 |
| **可重复调用** | ❌ 只能调用一次 | ✅ 可多次调用 |
| **底层原理** | 调用native start0() | 普通方法调用 |

### 3.2 代码演示

```java
public class StartVsRun {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            System.out.println("当前线程: " + Thread.currentThread().getName());
        });

        // 使用start() - 新线程执行
        thread.start();  // 输出: 当前线程: Thread-0

        // 使用run() - 主线程执行
        // thread.run();  // 输出: 当前线程: main
    }
}
```

### 3.3 为什么不能直接调用run()？

```java
// ❌ 错误做法：直接调用run()
thread.run();  // 只是普通方法调用，没有创建新线程

// ✅ 正确做法：调用start()
thread.start();  // 创建新线程，调用native start0()
```

## 四、sleep() vs wait()区别

### 4.1 核心对比表

| 对比项 | sleep() | wait() |
|--------|---------|--------|
| **所属类** | Thread类 | Object类 |
| **是否释放锁** | ❌ 不释放 | ✅ 释放 |
| **使用位置** | 任何地方 | synchronized块中 |
| **唤醒方式** | 超时自动唤醒 | notify() / notifyAll() |
| **响应中断** | ✅ 抛出InterruptedException | ✅ 抛出InterruptedException |
| **使用场景** | 暂停执行 | 线程间通信 |

### 4.2 代码演示

```java
public class SleepVsWait {
    private static final Object lock = new Object();

    public static void main(String[] args) {
        // sleep()示例
        Thread sleepThread = new Thread(() -> {
            synchronized (lock) {
                try {
                    System.out.println("sleep开始，持有锁");
                    Thread.sleep(2000);  // sleep不释放锁
                    System.out.println("sleep结束，释放锁");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });

        // wait()示例
        Thread waitThread = new Thread(() -> {
            synchronized (lock) {
                try {
                    System.out.println("wait开始，持有锁");
                    lock.wait();  // wait释放锁
                    System.out.println("wait结束，重新获取锁");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });

        sleepThread.start();
        waitThread.start();
    }
}
```

### 4.3 为什么wait()必须在synchronized块中调用？

```java
// ❌ 错误：直接调用wait()
// Object.wait();  // 抛出IllegalMonitorStateException

// ✅ 正确：在synchronized块中调用
synchronized (lock) {
    lock.wait();  // 正确，已持有锁
}

// 原因：wait()会释放锁，必须先持有锁才能释放
```

### 5.1 Thread常用方法

| 方法 | 说明 | 示例 |
|------|------|------|
| `start()` | 启动线程 | `thread.start()` |

| `currentThread()` | 获取当前线程 | `Thread.currentThread()` |

### 5.2 join()方法详解

```java
public class JoinDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("T1: " + i);
                try {
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });

        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("T2: " + i);
                try {
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });

        t1.start();
        t1.join();  // 主线程等待t1执行完毕
        
        t2.start();
        t2.join();  // 主线程等待t2执行完毕
        
        System.out.println("所有线程执行完毕");
    }
}
```

## 六、LeetCode题目解析

### 6.1 [1114. 按序打印（Easy）](https://leetcode.cn/problems/print-in-order/)

**题目描述**：
设计一个程序，让线程按顺序执行。有三个线程：A、B、C，分别调用 `first()`、`second()`、`third()`，输出 "first"、"second"、"third"。

**思路分析**：
使用 `synchronized` + `volatile` 标志位控制执行顺序。

```java
class Foo {
    private volatile int flag = 1;
    private final Object lock = new Object();

    public void first(Runnable printFirst) throws InterruptedException {
        synchronized (lock) {
            while (flag != 1) {
                lock.wait();
            }
            // 输出 first
            printFirst.run();
            flag = 2;
            lock.notifyAll();
        }
    }

    public void second(Runnable printSecond) throws InterruptedException {
        synchronized (lock) {
            while (flag != 2) {
                lock.wait();
            }
            // 输出 second
            printSecond.run();
            flag = 3;
            lock.notifyAll();
        }
    }

    public void third(Runnable printThird) throws InterruptedException {
        synchronized (lock) {
            while (flag != 3) {
                lock.wait();
            }
            // 输出 third
            printThird.run();
        }
    }
}
```

**复杂度分析**：
- 时间复杂度：O(1) 每次调用
- 空间复杂度：O(1)

## 七、今日学习要点总结

### 核心要点归纳

1. **线程创建方式**：Thread、Runnable、Callable、线程池，推荐使用线程池
2. **线程状态**：NEW → RUNNABLE → BLOCKED/WAITING/TIMED_WAITING → TERMINATED
3. **start() vs run()**：start()创建新线程，run()只是方法调用
4. **sleep() vs wait()**：sleep不释放锁，wait释放锁；sleep在Thread类，wait在Object类
5. **wait()必须在synchronized块中**：因为需要先获取锁才能释放锁
