## 一、泛型类型擦除原理

### 1.1 什么是类型擦除？

Java的泛型是**编译期特性**，不是运行时特性。在编译后，所有泛型信息都会被擦除，替换为原始类型（Raw Type）。

**类型擦除规则：**
- `List<String>` → `List`（无界泛型擦除为`Object`）
- `List<? extends Number>` → `List<Number>`（上界泛型擦除为上界类型）
- `List<? super Integer>` → `List<Integer>`（下界泛型擦除为下界类型）

### 1.2 类型擦除的代码演示

```java
import java.util.*;

public class TypeErasureDemo {
    public static void main(String[] args) {
        // 1. 编译后的字节码中，List<String> 和 List<Integer> 都是 List
        List<String> stringList = new ArrayList<>();
        List<Integer> intList = new ArrayList<>();
        
        // 运行时类型信息相同
        System.out.println(stringList.getClass() == intList.getClass()); // true
        System.out.println(stringList.getClass()); // java.util.ArrayList
        
        // 2. 反射验证：泛型信息只存在于编译期
        try {
            // 通过反射获取字段的泛型类型
            Field listField = MyContainer.class.getDeclaredField("list");
            Type genericType = listField.getGenericType();
            System.out.println("泛型类型: " + genericType); // java.util.List<java.lang.String>
            
            // 但运行时创建的实例不包含泛型信息
            MyContainer<String> container = new MyContainer<>();
            System.out.println("运行时类型: " + container.list.getClass()); // java.util.ArrayList
            
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

class MyContainer<T> {
    private List<T> list = new ArrayList<>();
}
```

### 1.3 类型擦除的影响

```java
public class ErasureImpact {
    // ❌ 编译错误：不能创建泛型数组
    // T[] array = new T[10];
    
    // ✅ 正确做法：通过反射创建
    @SuppressWarnings("unchecked")
    T[] createArray(Class<T> clazz, int size) {
        return (T[]) Array.newInstance(clazz, size);
    }
    
    // ❌ 编译错误：不能用instanceof检查泛型类型
    // if (obj instanceof List<String>) { }
    
    // ✅ 正确做法：检查原始类型
    if (obj instanceof List) {
        List<?> list = (List<?>) obj;
    }
    
    // ❌ 编译错误：不能创建泛型实例
    // T instance = new T();
    
    // ✅ 正确做法：通过Supplier或Class创建
    <T> T createInstance(Supplier<T> supplier) {
        return supplier.get();
    }
}
```

### 1.4 桥接方法（Bridge Method）

类型擦除后，Java编译器会自动生成桥接方法来保证多态性：

```java
public class Node<T> {
    public T data;
    
    public void setData(T data) {
        this.data = data;

}

public class StringNode extends Node<String> {

    }
    
    // 编译器自动生成的桥接方法（字节码中存在）
    // public void setData(Object data) {
    //     setData((String) data);
    // }
}
```

## 二、通配符PECS（Producer Extends, Consumer Super）

### 2.1 PECS原则详解

| 场景 | 通配符 | 含义 | 使用场景 |
|------|--------|------|---------|
| Producer（生产者） | `? extends T` | 只读取，不写入 | 从集合中获取数据 |
| Consumer（消费者） | `? super T` | 只写入，不读取 | 向集合中添加数据 |
| 既读又写 | `<T>` | 无通配符 | 读写都用同一类型 |

### 2.2 `? extends T`（上界通配符）

```java
public class UpperBoundDemo {
    
    // ✅ 可以读取（返回T或T的子类）
    public static double sumOfList(List<? extends Number> list) {
        double sum = 0;
        for (Number num : list) {
            sum += num.doubleValue();
        }
        return sum;
    }
    
    // ❌ 不能写入（编译器不知道具体类型）
    public static void addNumber(List<? extends Number> list) {
        // list.add(1);      // 编译错误
        // list.add(1.0);    // 编译错误
        // list.add(1L);     // 编译错误
        // 原因：编译器只知道list是Number的某个子类的列表
        // 可能是Integer List，也可能是Double List
    }
    
    public static void main(String[] args) {
        List<Integer> intList = Arrays.asList(1, 2, 3);
        List<Double> doubleList = Arrays.asList(1.5, 2.5, 3.5);
        
        // ✅ 都可以传入
        System.out.println(sumOfList(intList));   // 6.0
        System.out.println(sumOfList(doubleList)); // 7.5
    }
}
```

**为什么不能写入？**

```java
// 假设允许写入
List<? extends Number> list = new ArrayList<Integer>();
list.add(1.5);  // ❌ 如果允许，这行代码会把Double放入Integer List

// 编译器的逻辑：
// 1. list可能是List<Integer>
// 2. 1.5是Double类型
// 3. Double不是Integer的子类
// 4. 类型不安全，拒绝编译
```

### 2.3 `? super T`（下界通配符）

```java
public class LowerBoundDemo {
    
    // ✅ 可以写入（T或T的子类都可以）
    public static void addNumbers(List<? super Integer> list) {
        list.add(1);
        list.add(2);
        list.add(3);
        // list.add(1.5);  // 编译错误：Double不是Integer的子类
    }
    
    // ✅ 可以读取（但只能作为Object读取）
    public static void printNumbers(List<? super Integer> list) {
        for (Object obj : list) {
            // 只能用Object类型接收
            System.out.println(obj);
        }
    }
    
    public static void main(String[] args) {
        List<Number> numberList = new ArrayList<>();
        List<Object> objectList = new ArrayList<>();
        
        // ✅ 都可以传入
        addNumbers(numberList);
        addNumbers(objectList);
    }
}
```

**为什么可以写入？**

```java
// 假设list是List<Number>或List<Object>
List<? super Integer> list = new ArrayList<Number>();
list.add(1);  // ✅ Integer是Number的子类，可以放入Number List

// 编译器的逻辑：
// 1. list至少是List<Integer>（下界保证）
// 2. Integer可以放入List<Integer>
// 3. 类型安全，允许编译
```

### 2.4 PECS经典应用：Collections.copy

```java
public class PecsExample {
    
    // Collections.copy的签名
    // public static <T> void copy(List<? super T> dest, List<? extends T> src)
    
    public static void main(String[] args) {
        List<Integer> src = Arrays.asList(1, 2, 3);
        List<Number> dest = new ArrayList<>(Arrays.asList(0, 0, 0));
        List<Object> dest2 = new ArrayList<>(Arrays.asList(0, 0, 0));
        
        // ✅ src是Producer（提供Integer），用extends
        // ✅ dest是Consumer（接收Integer），用super
        Collections.copy(dest, src);   // OK
        Collections.copy(dest2, src);  // OK
    }
}
```

### 2.5 PECS速记口诀

```
PECS = Producer Extends, Consumer Super
      ↑               ↑
      从集合中取      往集合中放
      
读取用 extends（上界）
写入用 super（下界）
既读又写用具体类型（无通配符）
```

## 三、checked vs unchecked异常

### 3.1 异常体系结构

```
Throwable
├── Error（系统级错误，不应捕获）
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── VirtualMachineError
│
└── Exception
    ├── RuntimeException（unchecked，运行时异常）
    │   ├── NullPointerException
    │   ├── ArrayIndexOutOfBoundsException
    │   ├── ClassCastException
    │   ├── IllegalArgumentException
    │   ├── NumberFormatException
    │   ├── ConcurrentModificationException
    │   └── ...
    │
    └── 其他Exception（checked，编译时异常）
        ├── IOException
        ├── SQLException
        ├── ClassNotFoundException
        ├── FileNotFoundException
        ├── InterruptedException
        └── ...
```

### 3.2 Checked Exception（编译时异常）

**特点：**
- 编译器强制要求处理（try-catch或throws）
- 通常是外部因素导致的异常
- 调用者必须知道并处理

```java
public class CheckedExceptionDemo {
    
    // 编译时异常：必须处理
    public void readFile(String path) throws IOException {
        FileInputStream fis = new FileInputStream(path); // 编译错误如果不处理
        BufferedReader reader = new BufferedReader(new InputStreamReader(fis));
        String line = reader.readLine();
    }
    
    // ✅ 方式1：try-catch处理
    public void processFile(String path) {
        try {
            readFile(path);
        } catch (IOException e) {
            System.err.println("文件读取失败: " + e.getMessage());
        }
    }
    
    // ✅ 方式2：throws向上抛出
    public void processData(String path) throws IOException {
        readFile(path);
    }
}
```

### 3.3 Unchecked Exception（运行时异常）

**特点：**
- 编译器不强制处理
- 通常是编程错误导致的异常
- 应该在代码中避免，而不是捕获

```java
public class UncheckedExceptionDemo {
    
    // 运行时异常：不强制处理，但可能发生
    public int divide(int a, int b) {
        return a / b; // 可能抛出 ArithmeticException
    }
    
    public String getArrayElement(String[] arr, int index) {
        return arr[index]; // 可能抛出 ArrayIndexOutOfBoundsException
    }
    
    public void processList(List<String> list) {
        for (String s : list) { // 可能抛出 NullPointerException
            System.out.println(s.length());
        }
    }
    
    // ✅ 最佳实践：防御性编程，而不是捕获运行时异常
    public int safeDivide(int a, int b) {
        if (b == 0) {
            throw new ArithmeticException("除数不能为零");
        }
        return a / b;
    }
}
```

### 3.4 常见RuntimeException速查表

| 异常类 | 常见原因 | 防御方法 |
|--------|---------|---------|
| `NullPointerException` | 调用null对象的方法或属性 | 判空检查，使用Optional |
| `ArrayIndexOutOfBoundsException` | 数组索引越界 | 检查索引范围 |
| `ClassCastException` | 强制类型转换失败 | 使用instanceof检查 |
| `IllegalArgumentException` | 方法参数非法 | 参数校验 |
| `NumberFormatException` | 字符串转数字失败 | 使用正则或try-catch |
| `ConcurrentModificationException` | 集合并发修改 | 使用迭代器或并发集合 |
| `UnsupportedOperationException` | 不支持的操作（如不可变集合） | 使用可变集合 |
| `StackOverflowError` | 递归过深 | 优化递归或改用迭代 |

## 四、try-with-resources原理

### 4.1 基本语法

```java
// Java 7+
try (Resource resource = new Resource()) {
    // 使用资源
} // 自动调用resource.close()
```

### 4.2 AutoCloseable接口

```java
// AutoCloseable是try-with-resources的基础
public interface AutoCloseable {
    void close() throws Exception;
}

// Closeable是AutoCloseable的子接口（更严格）
public interface Closeable extends AutoCloseable {
    @Override
    void close() throws IOException; // 只抛IOException
}
```

### 4.3 自定义AutoCloseable资源

```java
public class DatabaseConnection implements AutoCloseable {
    private String connectionId;
    private boolean isOpen;
    
    public DatabaseConnection(String connectionId) {
        this.connectionId = connectionId;
        this.isOpen = true;
        System.out.println("打开数据库连接: " + connectionId);
    }
    
    public void query(String sql) {
        if (!isOpen) {
            throw new IllegalStateException("连接已关闭");
        }
        System.out.println("执行查询: " + sql);
    }
    
    @Override
    public void close() {
        isOpen = false;
        System.out.println("关闭数据库连接: " + connectionId);
    }
}

// 使用
public class TryWithResourcesDemo {
    public static void main(String[] args) {
        try (DatabaseConnection conn = new DatabaseConnection("conn-1")) {
            conn.query("SELECT * FROM users");
            conn.query("SELECT * FROM orders");
        } // 自动调用close()
    }
    
    // 输出：
    // 打开数据库连接: conn-1
    // 执行查询: SELECT * FROM users
    // 执行查询: SELECT * FROM orders
    // 关闭数据库连接: conn-1
}
```

### 4.4 多资源管理

```java
public class MultiResourceDemo {
    public static void main(String[] args) {
        try (
            FileInputStream fis = new FileInputStream("input.txt");
            FileOutputStream fos = new FileOutputStream("output.txt");
            BufferedReader reader = new BufferedReader(new InputStreamReader(fis))
        ) {
            String line;
            while ((line = reader.readLine()) != null) {
                fos.write(line.getBytes());
            }
        } // 按声明的逆序调用close()
        // close顺序: reader → fos → fis
    }
}
```

### 4.5 异常抑制（Exception Suppression）

```java
public class ExceptionSuppressionDemo {
    public static void main(String[] args) {
        Exception suppressed = null;
        
        try (MyResource resource = new MyResource()) {
            throw new RuntimeException("主异常");
        } catch (Exception e) {
            suppressed = e;
            System.out.println("捕获异常: " + e.getMessage());
            
            // 获取被抑制的异常
            Throwable[] suppressedExceptions = e.getSuppressed();
            for (Throwable t : suppressedExceptions) {
                System.out.println("被抑制的异常: " + t.getMessage());
            }
        }
    }
}

class MyResource implements AutoCloseable {
    @Override
    public void close() {
        throw new RuntimeException("关闭时的异常");
    }
}

// 输出：
// 捕获异常: 主异常
// 被抑制的异常: 关闭时的异常
```

### 4.6 try-with-resources的实际应用

```java
public class PracticalExamples {
    
    // 示例1：文件操作
    public void processFile(String inputPath, String outputPath) throws IOException {
        try (
            BufferedReader reader = Files.newBufferedReader(Path.of(inputPath));
            BufferedWriter writer = Files.newBufferedWriter(Path.of(outputPath))
        ) {
            String line;
            while ((line = reader.readLine()) != null) {
                writer.write(line.toUpperCase());
                writer.newLine();
            }
        }
    }
    
    // 示例2：数据库操作（伪代码）
    public List<User> queryUsers() throws SQLException {
        try (
            Connection conn = dataSource.getConnection();
            PreparedStatement stmt = conn.prepareStatement("SELECT * FROM users");
            ResultSet rs = stmt.executeQuery()
        ) {
            List<User> users = new ArrayList<>();
            while (rs.next()) {
                users.add(new User(rs.getString("name")));
            }
            return users;
        }
    }
    
    // 示例3：网络连接
    public String fetchData(String url) throws IOException {
        try (InputStream is = new URL(url).openStream()) {
            return new String(is.readAllBytes());
        }
    }
}
```

## 五、LeetCode题目解析

### [20. 有效的括号](https://leetcode.cn/problems/valid-parentheses/)（Easy）

**题目：** 给定一个只包含 `()`、`[]`、`{}` 的字符串，判断字符串是否有效。

**思路：** 使用栈匹配括号

```java
import java.util.*;

class Solution {
    public boolean isValid(String s) {
        Stack<Character> stack = new Stack<>();
        
        for (char c : s.toCharArray()) {
            if (c == '(' || c == '[' || c == '{') {
                stack.push(c);
            } else {
                if (stack.isEmpty()) {
                    return false;
                }
                
                char top = stack.pop();
                if (c == ')' && top != '(') return false;
                if (c == ']' && top != '[') return false;
                if (c == '}' && top != '{') return false;
            }
        }
        
        return stack.isEmpty();
    }
}
```

**复杂度分析：**
- 时间复杂度：O(n)，遍历一次字符串
- 空间复杂度：O(n)，最坏情况全部入栈

### [155. 最小栈](https://leetcode.cn/problems/min-stack/)（Medium）

**题目：** 设计一个支持 `push`、`pop`、`top` 和 `getMin` 操作的栈。

**思路：** 使用辅助栈维护最小值

```java
import java.util.*;

class MinStack {
    private Stack<Integer> stack;
    private Stack<Integer> minStack;
    
    public MinStack() {
        stack = new Stack<>();
        minStack = new Stack<>();
    }
    
    public void push(int val) {
        stack.push(val);
        if (minStack.isEmpty() || val <= minStack.peek()) {
            minStack.push(val);
        }
    }
    
    public void pop() {
        int val = stack.pop();
        if (val == minStack.peek()) {
            minStack.pop();
        }
    }
    
    public int top() {
        return stack.peek();
    }
    
    public int getMin() {
        return minStack.peek();
    }
}
```

**复杂度分析：**
- 时间复杂度：O(1) for all operations
- 空间复杂度：O(n)

## 六、今日学习要点总结

### 6.1 泛型核心要点

1. **类型擦除**：编译后泛型信息消失，运行时只有原始类型
2. **PECS原则**：
   - `? extends T`：只读（Producer）
   - `? super T`：只写（Consumer）
   - `<T>`：既读又写
3. **泛型限制**：不能创建泛型数组、不能用instanceof检查泛型类型、不能创建泛型实例

### 6.2 异常核心要点

1. **Checked Exception**：编译时强制处理，通常是外部因素
2. **Unchecked Exception**：运行时异常，编程错误导致，应该避免而非捕获
3. **try-with-resources**：自动关闭资源，按声明逆序关闭，支持异常抑制
